SAFR — Руководство пользователя

SAFR (Signal Analysis & Filtering Research) — настольное приложение для исследования эффективности методов фильтрации шумов и восстановления сигналов малой амплитуды. Программа позволяет моделировать синтетические сигналы, добавлять к ним различные типы шумов, применять методы фильтрации, измерять качество восстановления и экспортировать результаты.

1. Назначение программы

  • сравнение разных фильтров на одинаковом зашумлённом сигнале;
  • подбор параметров фильтра для заданного типа шума;
  • накопление серии экспериментов и выгрузка таблицы метрик;
  • подключение собственных реализаций сигналов и фильтров.

2. Запуск программы

python main.py

По умолчанию открывается GUI. При первом запуске автоматически создаются exports/, logs/, user_plugins/signals/, user_plugins/filters/.

Флаг --headless запускает самопроверку без окна — выводит список зарегистрированных компонентов.

3. Интерфейс в общих чертах

  • Левая колонка — выбор сигнала, шума, фильтра и кнопки.
  • Центральная область — вкладки Графики, Текст, Таблица.
  • Статус-бар — текущее состояние.

7. Настройка параметров

  • Дробные поля поддерживают научную запись (1e-5).
  • Стрелки меняют значение ~10 % от текущего.

8. Запуск моделирования

Кнопка Запустить моделирование. Расчёт идёт в фоне, окно остаётся отзывчивым. Результат — на вкладках «Графики», «Текст», «Таблица».

9. Чтение графиков

Три subplot: исходный, зашумлённый, восстановленный сигнал. В заголовках — SNR, ΔSNR, корреляция. Над канвой — панель matplotlib.

10. Чтение текстового отчёта

Вкладка «Текст» показывает параметры сигнала/шума/фильтра и метрики: MSE, RMSE, SNR до/после, ΔSNR, корреляция.

11. Чтение таблицы результатов

Вкладка «Таблица» накапливает все прогоны сессии. Очистка — кнопкой «Очистить результаты» или меню Файл → Очистить результаты.

12. Экспорт последнего эксперимента

Меню Файл → Экспортировать последний эксперимент…. Создаётся подпапка с файлами report.txt, report.xlsx (Parameters / Metrics / Signals), report.docx (с вставленными графиками).

13. Экспорт текущей сессии

Меню Файл → Экспортировать сессию (XLSX)…. Один лист Summary, одна строка = один эксперимент.

Экспорт сессии в DOCX и автоэкспорт после каждого прогона в MVP не реализованы.

14. Работа с пользовательскими плагинами

Один .py-файл в user_plugins/signals/ или user_plugins/filters/. Требования:

  • API_VERSION = "1.0";
  • PLUGIN — экземпляр наследника ISignal или IFilter из safr.plugins.api;
  • непустые name, description;
  • валидная params_schema.
Плагины выполняются в процессе приложения. Запускайте только доверенные файлы.

После добавления плагина — перезапустите приложение.

15. Ограничения текущей версии

  • Пользовательские плагины шумов не поддерживаются.
  • Вейвлет — только базис Хаара.
  • Butterworth — частотная маска (zero-phase), не IIR.
  • Фильтр Калмана — упрощённая 1D-модель.
  • Автоэкспорт после каждого прогона отсутствует.
  • Пакетный режим и Монте-Карло не реализованы.
  • Только 1D-сигналы, равномерная дискретизация.
  • Максимум 10 000 000 отсчётов на сигнал.
  • Интерфейс только на русском; тёмной темы нет.

16. Типичные ошибки пользователя

СообщениеПричина
«frequency должна быть строго меньше Найквиста» Частота ≥ fs/2. Уменьшите частоту или увеличьте fs.
«width больше duration» Параметры импульса не помещаются в интервал.
«window_length должно быть нечётным» Окно фильтра должно быть 3, 5, 7, …
«cutoff должна быть строго меньше fs/2» Частота среза Butterworth превышает Найквист.
«мощность чистого сигнала равна нулю» Все параметры дают нулевой сигнал (напр. amplitude = 0).
«Нет данных для экспорта» Не запущен ни один эксперимент.
Плагин не появился в списке Нарушение контракта или файл не в нужном каталоге. Перезапустите.
Совет. Начинайте с параметров по умолчанию — они подобраны так, чтобы моделирование проходило корректно.
